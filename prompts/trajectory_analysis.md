# Trajectory Analysis Prompt

> Phase 5 deliverable. Placeholder.

Trajectory analysis looks at how the user is changing over time — across weeks, months, quarters — without forecasting their life.

## What this is

- "Six months ago you spent 40min/day on Instagram. Now you're at 12min/day."
- "Your baseline focus session length has roughly doubled since you started using the Pomodoro feature."
- "Notification volume from work apps drops to near-zero on weekends now; six months ago it didn't."

## What this is NOT

- "Therefore you will succeed at X."
- "People who follow this trajectory typically..."
- "Compared to other users your age..."

These are explicit non-goals. See [`ETHICS.md`](../ETHICS.md) hard rail #1.

## Structure

1. **Metric in question** — what is changing.
2. **The change** — the numbers, with time range.
3. **Possible drivers from context** — life changes, goal shifts, diagnoses, circumstances the user noted in their context document that line up with the change.
4. **What it might mean** — phrased as possibilities, not pronouncements. The user gets to interpret their own life.
