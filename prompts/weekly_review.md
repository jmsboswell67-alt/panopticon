# Weekly Review Prompt

> Phase 5 deliverable. Placeholder.

The weekly review ritual generates an insight covering the past 7 days of behavioral data, framed by the user's context and goals.

## Inputs

- The user's context document (relevant fields, not the whole thing).
- This week's behavioral summary: focus minutes per app, notification volume, sleep approximation, screen-on session counts, deviations from the user's own baseline.
- The user's stated values and current goals.
- The selected persona (default: Gudea).

## Structure

1. **Observation** — what the data shows, plainly.
2. **Framing** — how that compares to the user's own baseline (NOT to other people).
3. **Connection to stated values/goals** — gaps, alignments, surprises.
4. **One question** — leave the user with something to think about, not a prescription.

## Constraints

- Inherits all rules from [`trauma_informed_base.md`](trauma_informed_base.md).
- Length: ~300-500 words. Long enough to be substantive, short enough to actually read.
- Cite supporting metrics. The "show your work" UI lets the user click any claim and see the underlying numbers.
